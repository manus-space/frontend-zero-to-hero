# Task 05: Login / Register Form Layout

## Introduction
Forms are how users interact with your backend. As a Spring Boot developer, you'll eventually connect these inputs to a REST API.

## Learning Points
- **Form Elements**: Using `<form>`, `<label>`, and `<input>` correctly.
- **Input Types**: Understanding the difference between `type="email"`, `type="password"`, etc.
- **Label + Input Pairing**: Why the `for` and `id` attributes are essential for accessibility.
- **Basic Validation UI**: Using CSS pseudo-classes like `:invalid` and `:focus` to give users feedback.

## Why this matters for SaaS
Every SaaS needs an authentication system. A clean, user-friendly login form reduces friction and makes your app feel more professional.

## How to run
1. Open `index.html`.
2. Try typing an invalid email (without @) and see how the border turns red (if configured).
3. Notice how clicking the label automatically focuses the corresponding input.
