# Task 02: Responsive Navigation Bar

## Introduction
A navigation bar is the most common component in any web application. It helps users find their way around.

## Learning Points
- **Flexbox**: Using `justify-content: space-between` to push the logo to one side and links to the other.
- **Media Queries**: Changing the layout based on screen size.
- **Mobile-First Thinking**: Planning how the UI looks on small screens.
- **CSS-Only Hamburger Menu**: Using the "Checkbox Hack" to toggle a menu without writing any JavaScript.

## Why this matters for SaaS
Navigation is critical for SaaS. You need a place for users to switch between "Features", "Pricing", and "Login". A responsive navbar ensures your SaaS works on mobile phones too.

## How to run
1. Open `index.html`.
2. Resize your browser window. Watch how the links disappear and the "hamburger" icon appears.
3. Click the hamburger icon to see the mobile menu toggle.
