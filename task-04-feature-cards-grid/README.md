# Task 04: Feature Cards Grid

## Introduction
SaaS products often have many features. The best way to show them is in a grid of "cards".

## Learning Points
- **CSS Grid**: Learning how to create a flexible, multi-column layout with very few lines of code.
- **Card Styling**: Using `box-shadow` and `border-radius` to create modern, clean UI components.
- **Hover States**: Making the UI feel alive with `transform` animations.
- **Responsive Columns**: Using `auto-fit` and `minmax` so the grid automatically adjusts for mobile and desktop without needing many media queries.

## Why this matters for SaaS
Almost every SaaS homepage uses feature cards to explain the value proposition. Mastering the grid layout is essential for any frontend developer.

## How to run
1. Open `index.html`.
2. Hover over each card to see the "lifting" effect.
3. Try resizing the browser window to see the cards wrap into fewer columns.
