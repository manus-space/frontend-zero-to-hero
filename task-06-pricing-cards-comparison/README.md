# Task 06: Pricing Cards Comparison

## Introduction
How do you sell your SaaS? You show the price! A comparison table helps users decide which plan fits them best.

## Learning Points
- **Multi-column Layout**: Using Flexbox to align cards side-by-side.
- **Popular Badge**: Using `position: absolute` and `transform` to place a badge on top of a card.
- **Price Highlighting**: Using font sizes and weights to make the most important information (the price) stand out.
- **Visual Hierarchy**: Using a "Primary" button for the plan you want users to buy.

## Why this matters for SaaS
The pricing page is often the final step in a user's journey. Good design here can directly increase your revenue.

## How to run
1. Open `index.html`.
2. Notice how the "Pro" plan is slightly larger and has a "Most Popular" badge.
3. Observe how the buttons are pushed to the bottom of the cards even if the text length varies (using `flex-grow`).
